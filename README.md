## 프로젝트 개요

이 저장소는 리눅스에서 프로세스를 관리할 때 자주 사용하는 명령어인  
`top`, `ps`, `jobs`, `kill`의 **기본 개념, 사용 예시, 비교**를 정리한 과제용 자료입니다.

- 프로세스 상태를 실시간으로 확인하는 방법 이해
- 현재 실행 중인 프로세스를 조회하는 방법 학습
- 백그라운드 작업(job)을 다루는 방법 학습
- 프로세스를 안전하게 종료하는 방법 학습

> 코드 자체보다는, **명령어의 역할과 사용 방법을 문서화하는 것**에 중점을 둡니다.

## 사전 요구사항

이 문서를 따라 하기 위해서는 다음과 같은 환경이 필요합니다.

- 리눅스 환경 (또는 WSL / 가상머신 Ubuntu 등)
- 기본적인 터미널 사용법
- Bash 셸 사용(기본값이라면 추가 설정 불필요)

선택 사항:

- `htop` 등 추가 모니터링 도구를 사용해 본 경험 (비교용)

## 실행 환경 준비(설치 방법)

별도의 프로그램 설치 없이, 기본 리눅스 환경이면 대부분의 명령어가 포함되어 있습니다.

### 1) 리포지토리 클론(과제 제출용 예시)

```bash
git clone https://github.com/username/linux-process-commands.git
cd linux-process-commands
````

### 2) 명령어 존재 여부 확인

```bash
which top
which ps
type jobs
which kill
```

* `jobs`는 **bash 내장 명령어**이므로 `type jobs`로 확인할 수 있습니다.

## 명령어별 사용법

### 1. top – 실시간 시스템 모니터링

> CPU, 메모리 사용량 및 실행 중인 프로세스를 **실시간으로** 확인하는 명령어

#### 기본 사용

```bash
top
```

#### 화면 주요 항목

| 항목      | 설명          |
| ------- | ----------- |
| PID     | 프로세스 ID     |
| USER    | 프로세스 실행 사용자 |
| %CPU    | CPU 사용률     |
| %MEM    | 메모리 사용률     |
| TIME+   | 누적 실행 시간    |
| COMMAND | 실행된 프로그램 이름 |

#### 자주 사용하는 단축키

* `q` : top 종료
* `P` : CPU 사용량 기준 정렬
* `M` : 메모리 사용량 기준 정렬
* `k` : 특정 PID 프로세스 종료
* `h` : 도움말 표시

#### 예시 화면

![top example](https://upload.wikimedia.org/wikipedia/commons/6/6c/Top_Ubuntu.png)

---

### 2. ps – 현재 프로세스 목록 조회

> 실행 중인 프로세스를 **스냅샷 형태로 한 번만 출력**하는 명령어

#### 👀 기본 사용

```bash
ps
```

#### 전체 프로세스 상세 보기

```bash
ps aux
```

| 옵션 | 의미                     |
| -- | ---------------------- |
| a  | 모든 사용자 프로세스            |
| u  | 상세 정보(사용자, CPU, 메모리 등) |
| x  | 터미널 없이 실행 중인 프로세스 포함   |

#### 특정 프로그램 검색 예시

```bash
ps aux | grep python
```

---

### 3. jobs – 현재 셸의 작업 목록

> 현재 **셸에서 실행 중이거나 멈춘 작업(job)** 을 보여주는 명령어
> ※ bash 내장 명령어

#### 기본 사용

```bash
jobs
```

#### 백그라운드 실행 및 확인 예시

```bash
sleep 100 &
jobs
```

출력 예시:

```text
[1]+  Running    sleep 100 &
```

#### 관련 명령어

* `fg %1` : 1번 job을 포그라운드로 가져오기
* `bg %1` : 1번 job을 백그라운드에서 계속 실행

---

### 4. kill – 프로세스에 시그널 전송(종료)

> PID를 이용해 **종료 시그널 또는 기타 시그널**을 보내는 명령어

#### 기본 사용

```bash
kill PID
```

기본 시그널은 `SIGTERM(15)` 입니다.

#### 강제 종료 예시

```bash
kill -9 1234   # SIGKILL, 거의 항상 종료됨
```

#### 자주 사용하는 시그널

| 시그널 번호 | 이름      | 설명                 |
| -----: | ------- | ------------------ |
|      2 | SIGINT  | 인터럽트(Ctrl + C와 유사) |
|      9 | SIGKILL | 강제 종료(잡을 수 없음)     |
|     15 | SIGTERM | 정상 종료 요청(기본값)      |

## 명령어 비교표

| 명령어  | 주요 목적           | 동작 방식       | 대표 사용 예시              |
| ---- | --------------- | ----------- | --------------------- |
| top  | 실시간 시스템 상태 확인   | 인터랙티브(실시간)  | CPU·메모리 모니터링          |
| ps   | 프로세스 목록 한 번 출력  | 비인터랙티브(스냅샷) | `ps aux \| grep <이름>` |
| jobs | 현재 셸의 job 상태 확인 | 셸 내부 정보     | 백그라운드 작업 확인·제어        |
| kill | 프로세스에 시그널 전송    | PID 기반 제어   | 프로세스 종료·재시작 스크립트      |

## 자주 발생하는 문제와 해결 방법

> 실습 중 자주 만나는 오류를 간단히 정리했습니다.

### 1) `command not found`

```bash
top
bash: top: command not found
```

* 리눅스 표준 배포판에서는 거의 발생하지 않지만,
  **최소 설치 환경**에서는 패키지가 빠져 있을 수 있습니다.
* 해결: 패키지 관리자 이용

```bash
# Debian/Ubuntu
sudo apt install procps

# RHEL/CentOS
sudo yum install procps-ng
```

---

### 2) `jobs` 출력이 비어 있는 경우

* 현재 셸에서 **백그라운드로 실행 중인 작업이 없는 경우**입니다.
* 테스트 방법:

```bash
sleep 60 &
jobs
```

---

### 3) `kill`로 종료가 안 될 때

* `kill PID` 또는 `kill -15 PID`로도 종료되지 않으면,
  마지막 수단으로 **강제 종료** 시그널 사용:

kill -9 PID

> 단, 강제 종료는 데이터 유실 가능성이 있으므로
> **가능한 SIGTERM(15)을 먼저 사용**하는 것이 좋습니다.
